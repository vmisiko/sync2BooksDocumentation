eTIMS tracks inventory movements. Sync2Books exposes two operations:

- **Adjust** ‚Äî add or deduct stock for an item at a branch (`PUT ‚Ä¶/stock/adjust`)
- **Transfer** ‚Äî move stock between items/branches (`POST ‚Ä¶/stock/transfer`)

> **Stock applies to goods only.** Only **`GOODS`** items are stock-tracked ‚Äî for them, sales deduct inventory, so you must add stock before selling. **`SERVICE`** items are non-stock: these operations have no effect on them, and you never need to stock them to sell. See [Goods vs Services](/etims-catalog#goods-vs-services-stock-vs-non-stock).

Stock matters because, for goods, **sales deduct inventory** ‚Äî you must add stock before a good can be sold. See [Sales prerequisites](/etims-sales#prerequisites-read-this-first).

## Adjust stock

> Note the HTTP method is **`PUT`**, not `POST`.

```bash
curl -X PUT "https://api.sync2books.com/companies/{companyId}/integrations/etims/stock/adjust" \
  -H "X-API-Key: {apiKey}" \
  -H "Content-Type: application/json" \
  -d '{
    "itemId": "{itemId}",
    "branchId": "HQ",
    "quantity": 100,
    "action": "ADD"
  }'
```


### Request body

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `itemId` | string | ‚úÖ | the item `id` from Sync2Books |
| `branchId` | string | ‚úÖ | your branch key, e.g. `HQ` |
| `quantity` | number | ‚úÖ | amount to add or deduct |
| `action` | `ADD` \| `DEDUCT` | ‚úÖ | |
| `movementTypeCode` | string | ‚Äî | KRA stock movement type override |
| `referenceId` | string | ‚Äî | your own reference for traceability |

## Transfer stock

Move stock from one item/branch to another:

```bash
curl -X POST "https://api.sync2books.com/companies/{companyId}/integrations/etims/stock/transfer" \
  -H "X-API-Key: {apiKey}" \
  -H "Content-Type: application/json" \
  -d '{
    "itemId": "{fromItemId}",
    "fromBranchId": "HQ",
    "receivingItemId": "{toItemId}",
    "toBranchId": "BR-01",
    "quantity": 5
  }'
```

```json
{ "syncBatchId": "88bb‚Ä¶" }
```

### Request body

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| `itemId` | string | ‚úÖ | source item `id` |
| `fromBranchId` | string | ‚úÖ | source branch key |
| `receivingItemId` | string | ‚úÖ | destination item `id` |
| `toBranchId` | string | ‚úÖ | destination branch key |
| `quantity` | number | ‚úÖ | |
| `referenceId` | string | ‚Äî | your reference |

## Both are asynchronous

Adjust and transfer return a `syncBatchId` and are processed asynchronously ‚Äî track them the same way as everything else: [Tracking Results](/etims-tracking-results).

## Typical pattern

```
add stock      ‚Üí PUT  ‚Ä¶/stock/adjust   { action: "ADD",    quantity: 100 }
sell           ‚Üí POST ‚Ä¶/sales          (deducts inventory)
correct count  ‚Üí PUT  ‚Ä¶/stock/adjust   { action: "DEDUCT", quantity: 3 }
move branches  ‚Üí POST ‚Ä¶/stock/transfer { fromBranchId, toBranchId }
```
